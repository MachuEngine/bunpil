# CLAUDE.md — 분필 (사회 교사용 AI 어시스턴트)

> 에이전트 행동 규칙 + 프로젝트 컨텍스트.
> 상세 설계는 **DESIGN.md**, 빌드 절차는 **PROMPTS.md** 참고.
> 이 파일은 ~200줄 이하로 유지 (길어지면 준수율 하락).

## 프로젝트 한 줄

고등학교 사회 교사용 AI 어시스턴트. 두 모듈 — **출제(ReAct Agent)**, **생기부 윤문(검증 Chain)**. 포트폴리오 + 지인 교사 1인 실사용.

## 스택

- Python / FastAPI (비동기)
- LangGraph(출제 agent) / LCEL(생기부 chain)
- ChromaDB + BGE-reranker, 임베딩 BGE-M3 — **임베딩·리랭킹은 CPU**
- LLM: Qwen2.5 7B (vLLM, 배포는 RunPod 서버리스). GPT-3.5/Ollama는 **합성 데이터 평가 전용**
- UI: Gradio
- 배포: AWS EC2(앱) + RunPod 서버리스(GPU), Caddy HTTPS

## 핵심 컨벤션

- 모듈 분리: `common/rag`, `common/llm`, `modules/exam`, `modules/record`
- LLM 호출은 **추상화 레이어 경유** (백엔드 env로 전환: 로컬 ↔ RunPod)
- RAG: 영구 컬렉션 = 공개 자료만 / 업로드 지문 = **세션 임시 컬렉션 후 폐기**
- 시크릿은 `.env`(.gitignore), 커밋 금지. `.env.example` 유지

## 🔴 보안 하드룰 (예외 없음)

1. 실제 학생 데이터 미사용 — 전부 **합성/익명**
2. 개인정보 마스킹은 외부/모델 호출 **이전**에
3. 사용자 입력(생기부 메모·업로드 지문) **비저장**
4. 로그·캐시에 **PII 금지**
5. 생기부: 메모에 없는 사실 **추가 금지**(생성 아닌 '다듬기'). 출력에 교사 책임 고지

> 이 규칙은 모든 작업에 자동 적용. 편의를 위해서도 위반 금지. 위반 소지가 있으면 **멈추고 질문**.

## 행동 원칙 (Karpathy 4원칙 기반)

1. **코딩 전 사고** — 추측 금지. 가정은 명시하고, 해석이 여러 개면 나열한 뒤 물어볼 것. 막히면 멈추고 무엇이 불명확한지 말할 것.
2. **단순함 우선** — 문제를 푸는 최소 코드만. 요청하지 않은 기능·추상화·"유연성" 금지. (단, DESIGN.md가 명시한 구조는 정당한 요구사항이니 임의로 제거하지 말 것.)
3. **외과적 변경** — 변경한 모든 줄이 요청에 직결돼야 함. 요청 없는 기존 코드 삭제·리팩터 금지.
4. **목표 기반 실행** — 성공 기준을 정하고 검증될 때까지 루프. 각 작업은 완료 기준 통과 후 멈추고 보고.

> canonical Karpathy 파일을 쓰려면 위 '행동 원칙' 섹션을 그 파일 내용으로 교체.

## 작업 방식

- **PROMPTS.md**의 Phase 순서대로. 한 Phase = 한 커밋. 완료 기준 통과 후 다음.
- 새 환경변수가 생기면 `.env.example`에 추가.
- 배포(Phase 8)는 학습 목적 — 무엇을 왜 하는지 설명하며 진행.

## 워크플로우 프로토콜

모든 작업은 반드시 아래 순서를 따를 것:

1. **분석** — 변경 범위와 영향받는 모듈 파악
2. **설계** — 변경안 제시 후 승인 대기 (바로 구현 금지)
3. **구현** — 승인된 설계대로만
4. **검증** — `scripts/test_*.py` 실행 후 결과 보고
5. **완료 선언**

## 절대 금지 (가드레일)

- `mask_pii()` 호출 순서를 `polish()` 이후로 바꾸지 말 것
- `data/` 디렉토리에 실제 학생 데이터를 추가하지 말 것
- 로그에 사용자 입력 원문(메모·지문)을 출력하지 말 것
- `.env` 파일의 값을 코드에 하드코딩하지 말 것
- `rm -rf chroma_db/` 실행 금지
- 검증(`test_*.py`) 없이 완료 선언 금지

## 모듈별 주의사항

- **record 모듈**: `mask_pii → polish → validate` 순서 고정
- **exam 모듈**: LangGraph 노드 순서 변경 시 반드시 설계 단계 승인 받을 것
- **chroma_db**: 세션 임시 컬렉션은 반드시 `try/finally`로 폐기
- **runpod_handler**: 변경 시 로컬 Ollama로 먼저 검증 후 RunPod 적용
